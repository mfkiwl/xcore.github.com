Details:¶

XMOS is committed to collaborative, community development but not just in the software domain. We believe open source firmware development is ideally backed up by open and interoperable hardware development platforms. To this end, we would like to establish exactly such an open, low cost and modular hardware development system capable of supporting a wide range of projects and target applications. Not only that, but XMOS would also like to use the very same platform for the majority of our own development work on new software components, demos and perhaps even some reference designs.

We very much hope that the same basic platform can serve both XMOS own commercial requirements as well as underpinning a busy development community. To this end, we have proposed here a (very) rough outline of the goals of this new development system and we would like to have feedback, contributions and discussion from our community to make sure we realize an open development hardware standard that works, well enough, for everyone. Probably we can’t please all of the people all of the time but lets see how close we can get. Ultimately the platform needs to serve a variety of XMOS commercial development needs but at this point we don’t see any reason why these can’t co-exist with the goals of the wider community.

So this repository is the forum to develop this open standard for a modular XMOS development system.

Platform Overview
++++++++++++++

The basic proposal is for an XS1-L2 based, 4-layer “core” board with multiple application specific daughter boards that attach above(and maybe below) the core board, along the lines of arduino shields but with a much much larger amount of GPIO. For now I am calling the daughter cards Slices and the core boards Basekits. The naming derives from an XMOS internal PCB code naming scheme which has a cheesy theme.

The intention is that the slice boards should generally be realizable as simple 2-layer, hand assembled affairs, though of course there is no reason to preclude more complex designs. In most cases power can be provided to the daughter board from the core board. The Basekit board would be 4-layer, but users would be able purchase these from digikey for a very reasonable price.

In future additional Basekits could be offered, for example one having 2xL2 with decent interchip xlink bandwidth and supporting multiple side by side slices on top.

Connectors¶

This is a tricky one. The kind of vertical pass through connectors that create the best flexibility in the system topologies that can be created are very expensive, and above all we want this project to yield a low cost modular development system.

We could stick with the horizontal connectors used on the XK1, but that would result in a topology of ‘ugly protuberances’ which precludes the possibility of making compact prototypes that can be boxed up if required.

At the moment we are leaning towards a simple system having a base board with multiple slices connected above it. Many of these slices (like the ethernet for example) are going to be really tiny. Others would be defined by the size of the connectors on them (e.g. for an LCD slice).

Another option is to have short FFC cables and connectors to do the connections.

As you can probably tell by now, we’re looking for some input here...

Basekit
-----

We propose to use an L2 for the initial Basekit board. This is differentiated from the XK-1 because it offers the opportunity to simultaneously have Ethernet or USB networking capability, and one or more I/O devices, memory add-ons etc. For example, an Ethernet-Camera system, or a USB-LCD display, an Ethernet-USB bridge, etc. We think it is also beneficial to have a development platform with more processing and IO resources so that one can concentrate on debug and proof of concept and worry about optimisation later. For projects which wish to develop L1 based designs, it is straightforward to emulate an L1 using an L2, using either one core or a subset of threads on both cores. The XMOS tools roadmap is going to address this type of prototyping and quickstart activity.

As well as the vertical connectors for Slices, we propose to have XSYS connectors like on the XK-1 to chain multiple core boards (each with their own daughter boards) horizontally.

The current very rough L2 Basekit board proposal has:

  • A number of GPIO connectors to attach slices, with a gioven slice interfacing to core 0 GPIO or core 1 GPIO.
  • 2 XSYS connectors carrying JTAG and one 2 bit xlink.
  • clock, power supplies, reset, JTAG
  • 1 x XS1-L2

Please see the sample datasheet in this repo for more detail on proposed port maps and examples of how these map to various slice board functionality.

Slices¶

XMOS plans over the next year to produce a variety of Slice boards designed to facilitate our own applications development. Some of these are listed below, but bear in mind that this list is subject to change. While we hope and expect this set to be useful to community projects, we know this is only a start and we hope that our XCORE.com regulars will be producing their own daughter boards to supplement the functions provided by these XMOS offerings.

  • Ethernet Slice (10/100 phy, connector, magnetics).
  • USB Slice (SMSC USB phy, clock generation and USB-B connector) - there is a good argument for including this on the Basekit board though
  • Audio Slice (simple Stereo line level audio input and output with codecs and SPDIF optical output).
  • LCD Slice (QVGA LCD touchscreen connector, power supply for backlight)
  • Camera Slice (CMOS image sensor, mount, power supply)
  • Memory Slice (SDcard connector plus an SDRAM, SDCard not supplied)
  • Prototyping Slice (breadboard plus perhaps other bits and piece - LEDS, RS232 connector?)

While we can’t at this point put pricing on these, they are all pretty miniminal two layer boards with just the components listed above so you should be able to have a good guess - certainly many of them should be well into the sub $20 range.

Projects¶

As mentioned before hardware platform is intended to facilitate the development of interoperable open source SW components. To this end we have plans to get this process started by developing some showcase demos designed to run on different combos of Slicekit and the above slices, and which incorporate a variety of nascent component developments developed by XMOS and by the rest of the community. For example, we are considering the following:

Demo Base Slice 1 Slice 2 SW Components Description
Webserver L2 Ethernet NA Ethernet, TCPIP Simple webserver demo
MP3 player L2 Memory Audio SDCard R/W via SPI or SDIO, MP3 decoder, I2S, SPDIF FAT32/16 lib Play music files from SDCard.
QVGA MJPEG camera L2 Camera Ethernet Ethernet, JPEG encoder Motion JPEG stream over ethernet
SD filesystem Browser L2 LCD Memory SDRAM Controller, LCD driver, SDIO, jpeg decoder, graphics lib FAT32/16 lib Use touchscreen to browse SDcard filesystem, display pictures etc

We’d make an overall project repo for each of these, or for whichever showcases we settle on eventually. Additionally, each new Slice developed (either by XMOS or by anyone else) would get its own github repo.

Boxes¶

Because a profusion of small dev boards may present us with a bit of an inventory issue, we are thinking that we should manufacture and supply these boards in sets, which roughly align with the showcase demos. Currently the breakdown looks something like this:

Box Contents showcase demo
Starter Base-L2, Ethernet Slice, Prototyping Slice, XTAG2 Simple Webserver
Music Audio Slice, Memory Slice MP3 Player
Pictures LCD slice, Camera Slice MJPEG camera

We should aim to flesh out this list with more, or better ideas that may already be floating around that are in a more advanced state of development but are compatible with the base platform outlined here. Having said that, the Basekit board definition and the list of official Slice cards is for the time being up for discussion so nothing is set in stone yet.

Hardware Suppliers¶

XMOS is open to any ideas third party developers may have about what business models can be built around this proposal. The opportunity exists, since this is an open source initiative, for community members to build and supply any or all of the proposed boards to potential users regardless of whether XMOS originally designed them or not and whether or not XMOS is already supplying them. In fact we would not be averse to the idea of having the entire line supplied by members of the community if they can demonstrate the ability to fulfill demand.

Similarly, commercial products based on these modules are not precluded.
